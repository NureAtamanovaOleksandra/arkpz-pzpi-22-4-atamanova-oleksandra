Розробити документ Vision & Scope для програмної системи управління та автоматизації дистанційних замовлень у магазині.

ХІД РОБОТИ

У ході виконання лабораторної роботи було проведено детальне опрацювання концепції програмної системи управління та автоматизації дистанційних замовлень у магазині. Визначено актуальність проблеми, зокрема труднощі, з якими стикаються магазини під час обробки онлайн-замовлень, такі як низька автоматизація, ймовірність помилок та складність інтеграції з існуючими сервісами.
Було проаналізовано аналогічні рішення на ринку, такі як Rozetka та Prom.ua, що дозволило визначити унікальні переваги майбутньої системи. 
Сформульовано мету системи – спрощення процесів обробки замовлень, а також окреслено її функціонал, який включає автоматизацію процесів, інтеграцію з платіжними системами, трекінг замовлень та аналітику.
Було описано функціональність серверної частини, ІоТ клієнта, веб клієнта та мобільного клієнта.
Визначено програмні засоби та програмні інструменти, які будуть використані у проєкті.
Створений текст документа стосується програмної системи управління та автоматизації дистанційних замовлень у магазині та представлений у Додатку А.
 
ВИСНОВКИ

У результаті виконання лабораторної роботи було розроблено документ Vision & Scope для програмної системи управління та автоматизації дистанційних замовлень у магазині. Визначено актуальні проблеми, які система повинна вирішити, а також основні функціональні можливості, що дозволяють підвищити ефективність обробки замовлень. Детально описано вимоги до кожної частини системи.
 
ДОДАТОК А

VISION AND SCOPE DOCUMENT (КОНЦЕПЦІЯ ТА РАМКИ ПРОЄКТУ)

1 BUSINESS REQUIREMENTS (БІЗНЕС-ВИМОГИ)

1.1	Background (Передумови)

Програмна система управління та автоматизації дистанційних замовлень спрямована на вирішення проблем, які часто виникають у магазинах під час роботи з онлайн-замовленнями:
•	Довготривалий процес обробки замовлень.
•	Висока ймовірність помилок через ручний ввід даних.
•	Обмежена інтеграція з платіжними системами та іншими сервісами.
Метою є створення рішення, яке забезпечить автоматизацію цих процесів, знизить ризики помилок та оптимізує обслуговування клієнтів.

1.2	Business Opportunity (Бізнес-можливості)

Аналіз ринку показав наявність рішень від таких компаній, як Rozetka та Prom.ua. Проте вони:
•	Складні для інтеграції із середнім та малим бізнесом.
•	Мають високу вартість для базового функціоналу.
Запропонована система буде розроблена спеціально для одного магазину, з урахуванням його унікальних потреб:
•	Повна адаптація під конкретні бізнес-процеси магазину без зайвого функціоналу.
•	Простота у використанні, яка не вимагає значного навчання персоналу.
•	Оптимізація витрат завдяки фокусуванню на необхідному функціоналі без оплати сторонніх послуг.
2	VISION OF THE SOLUTION (КОНЦЕПЦІЯ РІШЕННЯ)

2.1	Vision Statement (Окреслення Концепції)

Система буде індивідуальним рішенням для автоматизації дистанційних замовлень, створених спеціально для потреб одного магазину, а саме магазину світлодіодної продукції, такої як стрічки, гексагони, кутові підсвітки і тд. Вона дозволить:
•	Ефективно обробляти замовлення від клієнтів.
•	Здійснювати швидкі та безпечні платежі.
•	Аналізувати показники продажів через зручні звіти.
Це рішення забезпечить простоту впровадження та використання, адаптоване до унікальних бізнес-процесів магазину, сприяючи його розвитку та підвищенню задоволеності клієнтів.

2.2	Major Features (Головна Функціональність)

1.	Автоматизована обробка замовлень: Забезпечує швидке та точне оформлення, опрацювання та передання замовлень у роботу без втручання оператора.
2.	Відстеження замовлень у реальному часі: Дозволяє клієнтам і магазинам бачити статус замовлення на кожному етапі – від оформлення до доставки.
3.	Аналітичні звіти: Система генерує звіти для оптимізації бізнес-процесів.
А саме:
•	Звіт про продажі – аналіз доходів, популярності товарів.
•	Звіт про клієнтів – відстеження частоти замовлень.
•	Звіт про транзакції – деталізація платіжних операцій, включаючи успішні та скасовані транзакції.
4.	Інтеграція з платіжними системами: Для забезпечення зручних і безпечних транзакцій.
3	SCOPE AND LIMITATIONS (РАМКИ ТА ОБМЕЖЕННЯ)

3.1	Scope of Initial Release (Рамки Первинного Випуску)

Первинний випуск системи буде спрямований на реалізацію базових функцій, які забезпечать ефективну роботу магазину світлодіодної продукції:
1.	Створення продукції магазину:
•	Додавання товарів.
•	Відображення товарів.
•	Автоматичне оновлення інформації про наявність товарів на основі замовлень.
2.	Реєстрація та авторизація користувачів:
•	Створення облікових записів для клієнтів.
•	Вхід в систему із захистом даних через паролі або інші методи авторизації.
3.	Створення замовлень:
•	Оформлення замовлень клієнтами через веб-інтерфейс.
4.	Автоматична обробка замовлень:
•	Автоматичне оновлення статусів замовлення (нове → в обробці → виконано).

3.2	Scope of Subsequent Releases (Рамки Наступних Випусків)

У наступних релізах планується розширення функціональних можливостей системи для забезпечення більш зручного використання та автоматизації:
1.	Мобільний додаток
•	Розробка мобільного додатка для клієнтів.
•	Можливість створення замовлень, перегляду їх статусів і отримання сповіщень через смартфон.
2.	Аналітика показників
•	Генерація звітів на основі наявних даних про продажі, клієнтів та транзакції.
3.	Відображення статусу замовлення через спеціальний пристрій:
•	Інтеграція пристрою індикації (наприклад, діоду), який в залежності від статусу замовлення буде горіти певним кольором.
4.	Інтеграція платіжної системи:
•	Додавання підтримки платіжної платформи, такої як Stripe.
5.	Система управління запасами:
•	Автоматичне оновлення наявності товарів у магазині на основі замовлень.

4	BUSINESS CONTEXT (БІЗНЕС КОНТЕКСТ)

4.3 Operating Environment (Робоче Середовище)

Система автоматизації дистанційних замовлень для магазину світлодіодної продукції (стрічки, гексагони, кутові підсвітки тощо) працюватиме в середовищі, яке забезпечує доступ через різні платформи — веб та мобільний додатки. Це дозволить забезпечити максимальну доступність і зручність для клієнтів, які використовують як настільні пристрої, так і мобільні телефони.
Для впровадження системи буде вибрано набір технологій, що забезпечить надійність, високу продуктивність та можливість масштабування:

Backend:
JavaScript обраний за свою гнучкість та широке застосування, що дозволяє швидко розробляти та впроваджувати рішення.
Основною платформою буде Node.js, оскільки вона забезпечує швидку обробку запитів і здатна ефективно працювати з великою кількістю одночасних з'єднань.
Для створення серверних додатків буде використано Express, легкий і гнучкий фреймворк, що спрощує процес розробки. 
Дані зберігатимуться в MongoDB, що дозволяє гнучко працювати з великими обсягами інформації, а для зручності взаємодії з базою даних використовується Mongoose, який спрощує роботу з MongoDB і полегшує валідацію даних. 
Для забезпечення безпеки користувачів і швидкої авторизації буде застосовуватись JWT (JSON Web Token) для роботи з токенами.

Frontend (Web):
Буде побудований за допомогою React, що дозволяє створювати високопродуктивні інтерфейси з великою екосистемою компонентів. 
Управління станом додатка буде реалізоване через Redux та Redux Toolkit, що забезпечить зручне і ефективне управління даними. 
Для роботи з серверними запитами та кешуванням даних буде використовуватись RTK Query, що значно покращує продуктивність. 
Formik та Yup допоможуть у створенні та валідації форм, а Material UI забезпечить стильний і зручний дизайн інтерфейсу за допомогою готових компонентів.

Mobile:
Мобільний додаток для платформи iOS та Android буде розроблений з використанням React Native, що дозволяє створювати кросплатформні додатки з мінімальними витратами часу та ресурсів. 
Для управління станом мобільного додатка буде використовуватись Redux та Redux Toolkit, а також Formik та Yup для створення та валідації форм.
Стилізація мобільного інтерфейсу здійснюватиметься за допомогою React Native Material, що дозволяє створювати сучасні та привабливі дизайни.

Інфраструктура системи буде базуватись на хмарній платформі, такої як AWS або Azure, що забезпечать високу доступність, надійність і масштабованість системи.
Також буде застосовано систему контролю версій Git та сервіс GitHub для управління кодом та співпраці над проєктом.
